# ADR-0001: 服务配置化、健康检查与优雅退出

**状态**：Accepted

## 背景
当前 3 个 Go 服务端口与 SQLite 路径硬编码，且缺少统一健康检查与优雅退出机制，不利于企业级部署与运维（环境差异、进程管理、可观测性）。

## 决策
1. **端口与 DB 路径配置化**：
   - 通过环境变量 `PORT` 和 `DB_PATH` 注入
   - 默认值保持与现状一致（8080/8081/8082 + 各自 `*.db`）
2. **健康检查**：每个服务新增
   - `GET /healthz`：返回 `status=ok`
   - `GET /readyz`：尝试 `db.Ping()`，用于容器/编排系统就绪探针
3. **优雅退出**：
   - 使用 `http.Server` + `Shutdown(timeout)`
   - 捕获 `SIGINT/SIGTERM`，避免直接杀进程导致请求中断或 db 资源未释放

## 影响
- API 新增 `healthz/readyz`，不破坏既有业务接口。
- 运维侧可通过环境变量统一配置，无需改代码。

## 备选方案
- 引入配置库（Viper/Koanf）与结构化日志（Zap），但会增加依赖与初始化复杂度；本阶段先采用最小改动的环境变量方案。
